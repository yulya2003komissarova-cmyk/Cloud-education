Раздел 1.1

Общие сведения об облачных сервисах

Крупные провайдеры облачных хранилищ объединяют тысячи серверов, расположенных в дата-центрах по всему миру. Именно они обеспечивают работу миллионов цифровых сервисов и приложений. Массовое распространение подобных решений объясняется ростом потребностей общества в удобном и безопасном доступе к информации, возможности редактировать данные с разных устройств, хранить объёмные файлы вне локальных накопителей, а также выполнять коллективную работу с документами.

Облачные технологии представляют собой распределённую систему обработки данных, в рамках которой пользователи получают доступ к вычислительным ресурсам, программам и сервисам через интернет в режиме реального времени. При этом физическое местоположение данных и самих серверов не имеет значения — достаточно любого устройства с подключением к сети и установленным браузером.

Модели обслуживания

🏗️

IaaS — Инфраструктура

Предоставление клиентам физических ресурсов — серверов, сетей и систем хранения данных — в форме арендуемых услуг. Подходит для развёртывания образовательных платформ и сложных исследовательских проектов.

🛠️

PaaS — Платформа

Пользователю предоставляется программная среда для разработки и запуска приложений. Включает промежуточное ПО, сервисы обмена сообщениями, интеграции и обработки данных.

📦

SaaS — ПО как сервис

Готовые приложения, развёрнутые в облаке. Пользователь работает через браузер или клиентский интерфейс, не устанавливая программы локально. Широко применяется для офисных пакетов и инструментов совместной работы.

Типы облачных решений

Частные (приватные) облака — внутренние инфраструктуры для использования в пределах одной организации. Обеспечивают высокий уровень контроля над данными. Администрирование может осуществляться самим предприятием или сторонней организацией.

Публичные (общедоступные) облака — формируются и эксплуатируются внешними поставщиками облачных услуг. Управление, масштабирование и обслуживание полностью входят в зону ответственности провайдера.

Гибридные облака — сочетают признаки частных и публичных моделей. Создаются для повышения гибкости ИТ-инфраструктуры и оптимизации затрат.

Раздел 1.2

Применение облачных технологий в образовательном процессе

В условиях цифровой трансформации облачные технологии приобретают всё большее значение как инструмент модернизации учебного процесса. Их внедрение соответствует требованиям ФГОС, ориентированных на повышение эффективности обучения, развитие цифровой грамотности и реализацию личностно ориентированного подхода.

Ключевое преимущество: учащиеся и преподаватели получают возможность взаимодействовать с образовательными ресурсами в любое время и из любой точки мира при наличии интернет-соединения.

Направления применения

Совместная работа. Платформы Google Workspace и Microsoft 365 дают возможность создавать интерактивные учебные материалы, проводить онлайн-тестирование, составлять расписания и вести коллективные базы данных.

Обучение программированию. Облачные IDE — GitHub Codespaces, Replit, Google Colab — позволяют выполнять лабораторные работы без установки ПО на личные устройства. Google Colab предоставляет среду для изучения Python и машинного обучения, а Replit поддерживает множество языков и удобен для групповой разработки.

Управление обучением (LMS). Moodle Cloud, Google Classroom и Microsoft Teams позволяют автоматизировать многие аспекты учебного процесса: размещение материалов, отслеживание успеваемости, организация обратной связи.

Интерактивность. Zoom, Microsoft Teams и LearningApps позволяют организовывать вебинары, интерактивные упражнения и игровые тесты.

Ограничения: зависимость от качества интернет-соединения, вопросы безопасности данных, необходимость обучения преподавателей работе с новыми инструментами.

Цель

Сформировать представление о линейных алгоритмах и научить составлять программы в облачных средах

Используемые облачные сервисы

💻

Replit

Облачная IDE для написания и запуска кода на Python

📝

Google Docs

Для совместного составления блок-схем и конспектов

📋

Google Forms

Для быстрой проверки знаний в конце урока

Ход урока

0–3 мин

Организационный момент

Приветствие, проверка готовности, объявление темы. Рассказ о преимуществах облачных сервисов: не нужно устанавливать ПО, можно работать из любой точки, легко делиться результатами.

3–8 мин

Актуализация знаний

Вопросы: Что такое алгоритм? Какие виды алгоритмов вы знаете? Примеры линейных действий в повседневной жизни.

8–18 мин

Теоретическая часть

Понятие линейного алгоритма, структура программы на Python (ввод → обработка → вывод), демонстрация программы в Replit, объяснение блок-схем.

18–38 мин

Практическая работа

Работа в группах по 2–3 человека. Задания: сумма двух чисел, площадь прямоугольника, перевод температуры °C → °F. Составление блок-схем в Google Docs.

38–43 мин

Проверка понимания

Тест в Google Forms: понятие линейного алгоритма, функции Python, результат выражения print(5+3*2), используемые сервисы.

43–45 мин

Подведение итогов

Рефлексия, оценка работы групп, домашнее задание — программа расчёта возраста через 10 лет в Replit.

Примеры кода

Python
# Простая программа приветствия name = input("Как тебя зовут? ") print("Привет, " + name + "!")
Python
# Задание 3: Перевод температуры celsius = float(input("Введите температуру в °C: ")) fahrenheit = celsius * 1.8 + 32 print("Температура в °F:", fahrenheit)
